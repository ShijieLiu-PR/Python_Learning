    面向对象：考虑问题要从对象的角度出发。
    主要思想：
        识别对象/分配职责/建立交互
        封装变化/隔离变化/执行变化
        封装  继承  多态
    特征：
        封装：
            1.数据角度：将多个基本类型合成一个自定义类型。
                    优势：复合人类的思考方式。
                    比如：学生，向量
            2.行为角度：对外提供简单的必要功能，隐藏实现细节。
                    优势：模块化开发，简化编程
            3.设计角度：
                分而治之：分解需求，让多个类协同完成。
                封装变化：每个变化点单独做成一个类。
                高内聚：类的内部处理一个变化点。
                低耦合：类与类的关系尽量做到互不影响。
        继承：重用现有类的功能与概念，并在此基础上进行扩展。
            作用：隔离用与做。
                 隔离客户端代码与实现方式。
        多态：调用父执行子。
            作用:通过重写执行不同变化点。
            比如：人调用交通工具的运输方法，执行的是飞机/火车的运输方法
                 图形管理器调用图形的计算面积的方法，执行的是圆形/矩形的计算面积方法。
    原则：
        开闭原则：增加新功能，不修改客户端代码。
                比如：某个技能，增加新影响效果，只要创建新的效果类，不需要修改其他类
        单一职责：每个类有且只有一个改变的原因。
                比如：技能系统中，每个类的职责明确。
        依赖导致：使用父类（抽象），而不是使用子类（具体）。
        组合复用：使用关联关系，代替继承关系。
                比如：技能释放器与影响效果，使用了关联关系。
        里式替换：父类出现的地方，都可以被子类替换。
                比如：技能释放器使用影响效果，但可以被所有具体效果替换。
        迪米特法则：低耦合
                比如：影响效果的变化，不影响释放器。
                    每种效果之间，互不影响。


多继承
    一个子类继承两个或者两个以上的基类，父类中的属性和方法同事存在同名方法的解析顺序（MRO， Method Resolution Order）：
    类自身 - 父类继承列表（由左向右） - 再上层父类

模块
    import
        1. 语法：
            import 模块名
            import 模块名 as 别名
        2. 作用：将某模块整体导入到当前模块中
        3. 使用：模块名.成员

    from import
        1. 语法：
            from  模块名 import 成员名 [as 别名1]
        2. 作用：将模块内的一个或者多个成员导入到当前模块的作用域中

    from import *
        1. 语法：from 模块名 import *
        2. 作用：将某模块的所有成员导入到当前模块中。
        3. 模块中以下划线（_）开头的属性，不会被导入，通常称这些成员为隐藏成员。
