面向对象 Object Oriented Programing
1. 概述
    面向过程：
        1. 分析出解决问题的步骤，然后逐步实现
            例如：婚礼筹办
                --发请帖（选照片、措辞、制作）
                --宴席（场地、找厨师、准备餐具、计划菜品）
                --婚礼仪式 （找婚庆公司、制定流程、提供设备、帮助执行）
        2. 公式：程序 = 算法 + 数据结构
        3. 优点：所有环节、细节字节掌握
        4. 缺点：考虑所有细节，工作量大
    面向对象：
        1.找人然后分配职责：考虑问题要从对象的角度出发。
        2.公式：程序 = 对象 + 交互
        3.优点：
            思想层面：更接近于人的思维方式，有利于梳理归纳、分析解决问题。
            技术层面：高复用、高扩展、高维护
2.类和对象
    类：一个抽象的概念，即生活中的类别
    对象：类的具体实例，即归属于某个类的个体
    类是创建对象的模板，包括数据成员和方法成员。
    类与类行为不同，对象与对象数据不同。
